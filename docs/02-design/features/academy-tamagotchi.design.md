> **Summary**: 학원 미션 기반 다마고치 육성 게이미피케이션 - 상세 설계

## 1. 설계 목표

- 학생이 직관적으로 다마고치를 육성하고 꾸밀 수 있는 UI 제공
- localStorage 기반으로 서버 없이 동작하는 완전한 클라이언트 앱
- CSS/SVG 기반 캐릭터 시스템으로 다양한 커스터마이징 지원
- 선생님(관리자) 모드로 미션 관리 및 인증 가능

### 설계 원칙

- 단순함: 초등~중등 학생도 바로 사용 가능한 UI
- 확장성: 아이템/미션을 쉽게 추가할 수 있는 데이터 구조
- 재미: 시각적 피드백과 애니메이션으로 성취감 제공

## 2. 아키텍처

### 2.1 시스템 구조

```
┌─────────────────────────────────────────────────┐
│                    Browser                       │
├─────────────────────────────────────────────────┤
│  index.html (SPA - 탭/섹션 전환 방식)            │
├──────────┬──────────┬──────────┬────────────────┤
│  홈/육성  │  미션    │  상점    │  경연대회       │
│  화면     │  목록    │  /꾸미기 │  /투표          │
├──────────┴──────────┴──────────┴────────────────┤
│              JavaScript Modules                  │
│  app.js │ tamagotchi.js │ mission.js │ shop.js   │
│  contest.js │ storage.js │ admin.js │ ui.js      │
├─────────────────────────────────────────────────┤
│              localStorage (JSON)                 │
│  학생 데이터 │ 다마고치 │ 미션 │ 아이템 │ 투표    │
└─────────────────────────────────────────────────┘
```

### 2.2 페이지 구조 (SPA 탭 방식)

```
index.html
├── [탭1] 내 다마고치 (홈)     - 캐릭터 표시, 상태, 레벨
├── [탭2] 미션                 - 미션 목록, 완료 처리
├── [탭3] 상점/꾸미기           - 아이템 구매, 외형 변경, 능력치 배분
├── [탭4] 경연대회              - 다마고치 갤러리, 투표
└── [관리자] 선생님 모드        - 미션 관리, 학생 관리, 간식상자
```

### 2.3 의존 관계

```
app.js ──→ ui.js (화면 전환)
  ├──→ tamagotchi.js (캐릭터 로직)
  ├──→ mission.js (미션 로직)
  ├──→ shop.js (상점 로직)
  ├──→ contest.js (경연대회 로직)
  ├──→ admin.js (관리자 로직)
  └──→ storage.js (모든 모듈이 사용)
```

## 3. 데이터 모델

### 3.1 localStorage 키 구조

| Key | 타입 | 설명 |
|-----|------|------|
| `atg_students` | Student[] | 학생 목록 |
| `atg_current_student` | string | 현재 로그인 학생 ID |
| `atg_missions` | Mission[] | 미션 목록 (선생님이 관리) |
| `atg_items` | Item[] | 아이템 카탈로그 |
| `atg_contest` | Contest | 현재 진행중인 경연대회 |
| `atg_admin_pin` | string | 관리자 PIN (기본: "0000") |

### 3.2 데이터 타입 정의

```javascript
// 학생 (Student)
{
  id: "std_001",              // 고유 ID
  name: "홍길동",              // 학생 이름
  createdAt: "2026-03-18",    // 생성일
  tamagotchi: {               // 다마고치 정보
    name: "뽀삐",             // 다마고치 이름
    level: 1,                 // 현재 레벨
    exp: 0,                   // 현재 경험치
    points: 0,                // 보유 포인트
    stats: {                  // 능력치
      str: 0,                 // 힘
      int: 0,                 // 지능
      cha: 0,                 // 매력
      sta: 0                  // 체력
    },
    appearance: {             // 외형
      bodyColor: "#FFD700",   // 몸 색상
      eyeStyle: "default",   // 눈 스타일
      hat: null,              // 모자 아이템 ID
      accessory: null,        // 악세서리 아이템 ID
      background: "default",  // 배경
      effect: null            // 특수 효과
    },
    ownedItems: []            // 보유 아이템 ID 목록
  },
  completedMissions: [],      // 완료한 미션 기록 [{missionId, completedAt, verifiedBy}]
  snackBoxChances: 0,         // 간식상자 기회 수
  streakDays: 0               // 연속 출석일
}

// 미션 (Mission)
{
  id: "msn_001",
  title: "오늘 학원에 왔어요",
  description: "출석 체크",
  type: "attendance",         // attendance | homework | exam | special
  exp: 10,                    // 보상 경험치
  repeatable: true,           // 매일 반복 가능 여부
  requireVerification: true,  // 선생님 인증 필요 여부
  verificationCode: null,     // 인증 코드 (선생님이 매일 변경)
  active: true                // 활성 상태
}

// 아이템 (Item)
{
  id: "item_001",
  name: "왕관",
  category: "hat",            // hat | accessory | background | effect | bodyColor | eyeStyle
  cost: 5,                    // 포인트 비용
  imageData: "crown",         // SVG 렌더링용 키
  unlockLevel: 1,             // 해금 필요 레벨
  description: "멋진 황금 왕관"
}

// 경연대회 (Contest)
{
  id: "contest_001",
  title: "3월 멋진 다마고치 대회",
  startDate: "2026-03-18",
  endDate: "2026-03-25",
  category: "beauty",         // beauty | power | popular
  votes: {},                  // { studentId: [voterStudentId, ...] }
  active: true
}
```

## 6. SVG 캐릭터 시스템

### 6.1 캐릭터 구조

캐릭터는 레이어 방식으로 SVG를 조합하여 렌더링한다.

```
레이어 순서 (아래→위):
1. 배경 (background)
2. 몸체 (body) - bodyColor 적용
3. 눈 (eyes) - eyeStyle 적용
4. 입 (mouth)
5. 모자 (hat) - 아이템
6. 악세서리 (accessory) - 아이템
7. 특수효과 (effect) - 아이템
```

### 6.2 SVG 렌더링 함수

```javascript
// renderTamagotchi(tamagotchi) → SVG 문자열 반환
// 각 레이어를 조합하여 하나의 SVG 생성
// 캐릭터 크기: 200x200px (반응형으로 확대/축소)
```

### 6.3 기본 캐릭터 파츠

| 파츠 | 종류 | 해금 조건 |
|------|------|-----------|
| 몸 색상 | 8색 (노랑, 빨강, 파랑, 초록, 분홍, 보라, 하늘, 주황) | Lv.4 + 각 3p |
| 눈 스타일 | 5종 (기본, 반짝, 졸림, 화남, 하트) | Lv.3 + 각 2p |
| 모자 | 6종 (왕관, 야구모자, 머리띠, 마법사, 산타, 꽃) | Lv.1~ + 각 3~5p |
| 악세서리 | 5종 (안경, 목걸이, 날개, 망토, 리본) | Lv.7~ + 각 5~10p |
| 배경 | 4종 (기본, 우주, 바다, 숲) | Lv.1~ + 각 8p |
| 특수효과 | 3종 (반짝이, 후광, 불꽃) | Lv.10~ + 각 15p |

## 7. 미션 인증 시스템

### 7.1 인증 방식

```
방식 1: 인증코드 입력 (기본)
  선생님이 매일 4자리 코드 생성 → 학생이 코드 입력 → 일치 시 완료

방식 2: 자동 반복 (출석 등)
  repeatable: true인 미션은 하루 1회 자동 리셋

방식 3: 선생님 직접 승인
  관리자 모드에서 학생별 미션 완료 처리
```

### 7.2 미션 완료 흐름

```
학생: 미션 선택 → 인증코드 입력 → [완료하기] 클릭
  │
  ├── 코드 일치 → 경험치 추가 → 레벨업 체크 → UI 업데이트
  │                              ├── 레벨업 → 축하 모달 + 보상 지급
  │                              └── 미달 → 경험치바만 업데이트
  │
  └── 코드 불일치 → "인증코드가 틀렸어요" 메시지
```

## 8. 간식상자 시스템

### 8.1 기회 획득 조건

| 조건 | 기회 수 |
|------|---------|
| 레벨 5 달성 | +1회 |
| 레벨 10 달성 | +2회 |
| 이후 5레벨마다 | +1회 |
| 연속 출석 7일 (체력 보너스) | +1회 |
| 경연대회 1등 | +1회 |

### 8.2 사용 흐름

```
학생: 간식상자 기회 보유 → 선생님에게 요청
선생님: 관리자 모드 → 해당 학생 → [사용 처리] 클릭
  → snackBoxChances -= 1
  → 실제 간식 지급 (오프라인)
```

## 9. 경연대회 시스템

### 9.1 대회 카테고리

| 카테고리 | 심사 기준 | 능력치 보너스 |
|----------|-----------|--------------|
| 멋진 외형 (beauty) | 투표수 | 매력(CHA)당 +0.5표 환산 |
| 파워 대결 (power) | STR 총합 | 힘(STR) 직접 비교 |
| 인기 대회 (popular) | 순수 투표수 | 없음 |

### 9.2 투표 규칙

- 1인당 대회당 최대 3표 (자기 자신 제외)
- 한 다마고치에 중복 투표 불가
- 대회 종료 후 결과 집계 및 순위 표시

### 9.3 Starter 레벨 한계 대응

> localStorage는 개인 브라우저에만 저장되므로, 경연대회는 다음과 같이 운영:

**방법 A: 발표 방식**
- 각 학생이 자기 다마고치 화면을 빔프로젝터로 보여줌
- 선생님이 관리자 모드에서 각 학생의 다마고치를 등록
- 학생들이 거수/스티커로 투표

**방법 B: 공용 PC 방식**
- 선생님 PC 1대에 모든 학생 데이터를 입력
- 학생들이 돌아가며 투표

**방법 C: 스크린샷 갤러리**
- 학생들이 다마고치 스크린샷을 찍어 제출
- 선생님이 갤러리에 등록 후 투표 진행
