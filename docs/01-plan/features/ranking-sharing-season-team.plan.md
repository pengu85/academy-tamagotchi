## 1. 개요

### 1.1 목적

학원 운영과 직접 연계되는 4개 기능을 추가하여, 다마고치 프로그램을 실제 수학학원 현장에서 활용할 수 있는 완성도 높은 도구로 만든다.

### 1.2 4개 기능 요약

| 기능 | 핵심 가치 | 학원 활용 |
|------|-----------|-----------|
| 주간 랭킹 보드 | 경쟁심 자극, 동기부여 | 학원 벽/빔프로젝터에 표시 |
| 학부모 공유 | 학부모 참여, 홍보 효과 | 카톡/SNS에 다마고치 카드 공유 |
| 시즌/이벤트 | 주기적 흥미 유지 | 시험기간 특별미션, 한정아이템 |
| 팀 미션 | 협동심, 반 단합 | 반 대항전, 팀 보상 |

## 2. 기능 1: 주간 랭킹 보드

### 2.1 요구사항

| ID | 요구사항 | 우선순위 |
|----|----------|:--------:|
| RK-01 | 레벨/총경험치/연속출석/미니게임 4가지 랭킹 기준 제공 | High |
| RK-02 | 주간 단위로 리셋 가능 (관리자 기능) | Medium |
| RK-03 | 1~3등 하이라이트 표시 (금/은/동 메달) | High |
| RK-04 | 관리자가 랭킹 보드를 프린트하기 좋은 레이아웃 제공 | Medium |

### 2.2 랭킹 기준

| 기준 | 정렬 방식 | 데이터 소스 |
|------|-----------|-------------|
| 레벨 | 레벨 내림차순 → 경험치 내림차순 | student.tamagotchi.level, exp |
| 총 경험치 | getTotalExpForLevel(level) + exp | 계산 값 |
| 연속 출석 | streakDays 내림차순 | student.streakDays |
| 미니게임 | perfectMinigames 내림차순 | student.perfectMinigames |

### 2.3 UI 위치

- 하단 탭에 추가하지 않음 (탭 과다 방지)
- **홈 화면에 "랭킹" 버튼** → 모달 or 별도 화면으로 표시
- 관리자 모드에서도 랭킹 확인 가능

## 3. 기능 2: 학부모 공유 (이미지 내보내기)

### 3.1 요구사항

| ID | 요구사항 | 우선순위 |
|----|----------|:--------:|
| SH-01 | 다마고치 상태 카드를 PNG 이미지로 저장할 수 있다 | High |
| SH-02 | 카드에 학생 이름, 다마고치, 레벨, 진화 단계, 능력치 표시 | High |
| SH-03 | 홈 화면에 "공유하기" 버튼 제공 | High |

### 3.2 카드 디자인

```
┌──────────────────────────────┐
│  🏫 ○○수학학원 다마고치       │
│                              │
│      [다마고치 SVG]           │
│                              │
│  뽀삐 (홍길동)               │
│  Lv. 10  |  청소년 단계       │
│                              │
│  💪3  🧠5  ✨2  ❤️4          │
│                              │
│  🔥 연속출석 15일             │
│  🏅 업적 8/14                │
└──────────────────────────────┘
```

### 3.3 기술 방식

- **Canvas API**: SVG → Canvas → PNG 변환
- `canvas.toDataURL('image/png')` → 다운로드 링크 생성
- 별도 라이브러리 없이 순수 JS로 구현

## 4. 기능 3: 시즌/이벤트 시스템

### 4.1 요구사항

| ID | 요구사항 | 우선순위 |
|----|----------|:--------:|
| EV-01 | 관리자가 이벤트를 생성/종료할 수 있다 | High |
| EV-02 | 이벤트 기간 중 EXP 배율을 설정할 수 있다 (1.5배, 2배) | High |
| EV-03 | 이벤트 전용 특별 미션을 추가할 수 있다 | Medium |
| EV-04 | 현재 이벤트가 홈 화면 상단에 배너로 표시된다 | High |
| EV-05 | 이벤트 한정 아이템을 설정할 수 있다 | Low |

### 4.2 이벤트 유형 (프리셋)

| 유형 | EXP 배율 | 설명 |
|------|:--------:|------|
| 시험대비 | 2.0x | 시험 기간 중 집중 학습 독려 |
| 방학특훈 | 1.5x | 방학 중 학원 출석 유지 |
| 자유이벤트 | 관리자 설정 | 커스텀 이벤트 |

### 4.3 데이터 구조

```javascript
// localStorage: atg_event
{
  id: "evt_001",
  title: "중간고사 대비 이벤트",
  type: "exam_prep",       // exam_prep | vacation | custom
  expMultiplier: 2.0,
  startDate: "2026-03-18",
  endDate: "2026-03-25",
  active: true,
}
```

## 5. 기능 4: 팀 미션 / 반 대항전

### 5.1 요구사항

| ID | 요구사항 | 우선순위 |
|----|----------|:--------:|
| TM-01 | 관리자가 팀을 생성하고 학생을 배정할 수 있다 | High |
| TM-02 | 팀별 총 경험치를 합산하여 순위를 표시한다 | High |
| TM-03 | 팀 대항전 기간을 설정할 수 있다 | Medium |
| TM-04 | 우승 팀 전원에게 보상(포인트/간식상자)을 지급한다 | Medium |

### 5.2 팀 구조

```javascript
// localStorage: atg_teams
[
  {
    id: "team_001",
    name: "A반",
    color: "#FF6B6B",
    members: ["std_001", "std_002", "std_003"],
  },
  ...
]

// localStorage: atg_team_battle
{
  title: "3월 반 대항전",
  startDate: "2026-03-18",
  endDate: "2026-03-31",
  active: true,
  teamReward: { points: 20, snackBox: 1 },
}
```

### 5.3 경험치 합산 방식

- 대항전 기간 중 각 학생이 획득한 경험치를 팀별로 합산
- `completedMissions` 에서 기간 내 기록 필터링하여 계산
